# Basic Algorithm Unit (BAU)

Minimal reference implementations of fundamental algorithms and data structures for competitive programming. Designed for quick review and template usage.

## 📚 Purpose

BAU serves as a **clean, minimal algorithm library** for:
- Quick review of core algorithmic concepts
- Copy-paste templates
- Study reference with working examples
- Foundation for solving complex CP problems

---

## 🗂️ Algorithm Categories

### **Sorting Algorithms**
- `QuickSort.cpp` - Partition-based sorting
- `MergeSort.cpp` - Divide and conquer sorting
- `HeapSort.cpp` - Heap-based sorting
- `ShellSort.cpp` - Gap sequence sorting

### **Data Structures**
- `DSU_DisjointSetUnion.cpp` - Union-Find with path compression
- `SegmentTree.cpp` - Range query tree (array-based)
- `SegmentTreeclass.cpp` - Range query tree (class-based)
- `RangeMaxQuery_BIT.cpp` - Fenwick Tree for range maximum
- `heap.cpp` - Min/Max heap operations
- `StackQueue.cpp` - Stack & Queue implementations
- `SinglyLinkedList.cpp` - Basic linked list
- `HashTable.cpp` - Simple chaining hash table (demo)

### **Tree Structures**
- `BinarySearchTree.cpp` - BST operations
- `AVL.cpp` - Self-balancing BST
- `SplayTree.cpp` - Self-adjusting BST
- `B-tree.cpp` - Multi-way search tree

### **Graph Algorithms**
- `Bellman-Ford.cpp` - Single-source shortest path (negative edges)
- `Prim.cpp` - Minimum Spanning Tree (edge-based)
- `Kruskal_MST.cpp` - Minimum Spanning Tree (DSU-based)
- `FindBridges.cpp` - Find bridges in undirected graph
- `ArticulationPoints.cpp` - Find cut vertices
- `TarjanSCC.cpp` - Strongly Connected Components
- `Dijkstra.cpp` - Single-source shortest paths (non-negative weights)
- `TopologicalSort.cpp` - Kahn's algorithm (topological ordering)

### **Advanced Techniques**
- `DifferenceArray.cpp` - 1D range update in O(1)
- `DifferenceArray2D.cpp` - 2D range update in O(1)
- `PrefixSum2D.cpp` - 2D range sum queries
- `Matrix.cpp` - Matrix operations

### **Utilities**
- `InOut.cpp` - Fast I/O templates

## 💡 Code Style

All implementations follow **competitive programming conventions**:
- Concise, contest-ready code
- Short variable names (`adj[]`, `f[]`, `low[]`, `num[]`)
- Minimal comments - focus on algorithm logic
- Fast I/O and efficient implementations
